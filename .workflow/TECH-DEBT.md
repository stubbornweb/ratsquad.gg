## P1 - Critical Issues

### DEBT-001: Framer Motion Not Used (ADR-001 Violation)
**Status:** Open
**Priority:** P1
**Category:** Architecture
**Impact:** Critical — violates the architectural decision that justified the Next.js migration
**Effort:** 4-6 hours

**Description:**
ADR-001 mandates "ALWAYS use Framer Motion for all animations — never use CSS-only animations." However, framer-motion is installed but not imported anywhere. All animations use manual IntersectionObserver with CSS classes (`fade-up`, `fade-in`, `.visible`).

**Files Affected:**
- `src/app/page.tsx` (lines 19-37 — IntersectionObserver, lines 40-120 — terminal typing)
- `src/app/roster/page.tsx` (lines 63-80 — IntersectionObserver)
- `src/app/globals.css` (lines 179-193 — CSS animation classes)

**Acceptance Criteria:**
- [ ] Replace IntersectionObserver with Framer Motion `useInView` or `whileInView`
- [ ] Replace CSS fade classes with Framer Motion `motion.div` + variants
- [ ] Replace terminal typing effect with React state-driven animation
- [ ] Remove unused `.fade-up`, `.fade-in`, `.visible` CSS classes
- [ ] framer-motion imported and used in all animated components

### DEBT-002: Monolithic Page Files
**Status:** Open
**Priority:** P1
**Category:** Architecture
**Impact:** High — files exceed 300-line limit, duplicated code, poor maintainability
**Effort:** 3-4 hours

**Description:**
`page.tsx` is 649 lines with all 8 sections inline. `roster/page.tsx` is 276 lines with duplicated navbar and footer. This violates the base rule "Keep files under 300 lines" and creates maintenance overhead.

**Files Affected:**
- `src/app/page.tsx` (649 lines)
- `src/app/roster/page.tsx` (276 lines)

**Proposed Structure:**
```
src/
  components/
    Navbar.tsx
    Footer.tsx
    sections/
      Hero.tsx
      About.tsx
      RosterPreview.tsx
      Join.tsx
      FAQ.tsx
      DiscordBanner.tsx
      TerminalHUD.tsx
```

**Acceptance Criteria:**
- [ ] Extract Navbar component (shared between pages)
- [ ] Extract Footer component (shared between pages)
- [ ] Extract each section into its own component
- [ ] page.tsx reduced to < 100 lines (composition only)
- [ ] roster/page.tsx reduced to < 100 lines
- [ ] No duplicated code between pages
- [ ] All functionality preserved
