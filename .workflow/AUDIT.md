## Structure Analysis

### File Organization
```
src/
  app/
    layout.tsx         (47 lines)  — Root layout with Google Fonts
    page.tsx           (649 lines) — Homepage, all sections inline
    globals.css        (1378 lines)— Global CSS with custom properties
    roster/
      page.tsx         (276 lines) — Roster page, duplicated navbar/footer
  lib/
    utils.ts           (6 lines)   — cn() utility (unused)
public/
  fonts/nastup/        — NASTUP brand font (WOFF2 + WOFF)
  *.svg                — Default Next.js placeholder assets
variant-v1/            — Legacy static HTML (archived)
docs/                  — Design specs and requirements
```

### Critical Structural Issues

**[P1] ADR Violations — Framer Motion Not Used**
- **ADR-001 mandates:** "ALWAYS use Framer Motion for all animations"
- **Actual:** All animations use manual IntersectionObserver + CSS classes (`fade-up`, `fade-in`)
- **Impact:** Violates the architectural decision that justified the Next.js migration
- **Effort:** 4-6h to migrate animations

**[P1] Monolithic Page Files**
- `page.tsx` is 649 lines with all sections inline (navbar, hero, about, roster, join, FAQ, discord, footer)
- `roster/page.tsx` is 276 lines with duplicated navbar and footer
- Violates base rule: "Keep files under 300 lines"
- **Effort:** 3-4h to extract components

**[P1] Massive globals.css (1378 lines)**
- Nearly all styling is custom CSS classes, not Tailwind utilities
- Defeats the purpose of having Tailwind CSS installed
- Contains duplicate `.footer-brand-mark` rule (lines 1232-1237 and 1240-1248)
- **Effort:** 8-12h for full Tailwind utility migration

## Code Quality Analysis

### TypeScript Usage
**Rating:** D (Minimal type safety)

**Issues:**

**[P2] No Custom Types or Interfaces**
- Roster data in `roster/page.tsx` is untyped array of objects
- FAQ data in `page.tsx` is untyped `{ q, a }` objects
- No `interface` or `type` definitions anywhere in the codebase
- **Recommendation:** Define `Member`, `FAQ`, `NavLink` interfaces

**[P2] `"use client"` on Every Page**
- Both pages are fully client-rendered with `"use client"`
- No server components used anywhere
- ADR-001 specifies ISR and server-side data fetching but none is implemented
- **Impact:** No SSR benefits, no ISR caching, larger client bundle

### React Patterns
**Rating:** C

**Issues:**

**[P2] Manual DOM Manipulation**
- `page.tsx:21-37` — Manual IntersectionObserver with `classList.add("visible")`
- `page.tsx:40-120` — Terminal typing effect using `innerHTML` directly
- `roster/page.tsx:63-80` — Duplicated IntersectionObserver
- **Impact:** Anti-pattern in React — should use state-driven rendering or Framer Motion
- **Risk:** innerHTML with static content is safe but sets a bad precedent

**[P3] Inline Styles**
- HUD stats grid uses inline `style={{}}` props (page.tsx:241-266)
- Roster header uses inline styles (roster/page.tsx:142)
- FAQ section uses inline margin overrides (page.tsx:471)
- **Recommendation:** Move to CSS classes or Tailwind utilities

**[P3] Duplicated Code Between Pages**
- Navbar: ~40 lines copy-pasted between page.tsx and roster/page.tsx
- Footer: ~60 lines copy-pasted between page.tsx and roster/page.tsx
- Scroll handler: ~5 lines duplicated
- IntersectionObserver: ~15 lines duplicated
- **Effort:** 2h to extract shared components

### CSS Implementation
**Rating:** C-

**Strengths:**
- Consistent design tokens via CSS custom properties
- Good responsive breakpoints at 768px and 992px
- clamp() used for responsive typography
- Custom scrollbar styling

**Issues:**

**[P2] CSS-First Instead of Tailwind-First**
- 1378 lines of custom CSS classes that could be Tailwind utilities
- Examples: `.section-container { max-width: 1200px; margin: 0 auto; }` could be `max-w-[1200px] mx-auto`
- Tailwind v4 is installed but barely leveraged
- Only `@import "tailwindcss"` and `@theme inline` are used from Tailwind

**[P3] Duplicate CSS Rule**
- `.footer-brand-mark` defined twice (lines 1232-1237 and 1240-1248)
- Second definition overrides first completely

**[P4] Utility Classes in CSS**
- `.text-center`, `.mx-auto`, `.block`, `.mt-2` defined manually (lines 94-97)
- These are built-in Tailwind utilities — redundant

## Performance Analysis

### Bundle & Rendering
**Rating:** C

**Issues:**

**[P2] Full Client-Side Rendering**
- Both pages are `"use client"` — no server-side rendering
- Content that could be static (FAQ, about text) is rendered client-side
- **Impact:** Slower First Contentful Paint, no SEO benefits from SSR

**[P3] Large CSS File**
- globals.css is 1378 lines loaded on every page
- No CSS splitting or critical CSS extraction
- Many classes only used on one page

**[P3] No Image Optimization**
- Hero background video referenced but file may not exist (`/assets/inspo-images/hero-bg.mp4`)
- No `next/image` usage for optimized image loading
- Default Next.js SVGs still in public/

### Strengths
- Standalone output configured for Vercel
- Google Fonts loaded via `next/font` (optimized)
- Font display: swap configured for NASTUP

## Security Analysis

**Rating:** B+ (Low attack surface)

**[P4] External Links Without rel Attributes**
- Discord links use `target="_blank"` but Next.js Link handles `rel` automatically
- Social links use `<a>` tags with `target="_blank"` — should add `rel="noopener noreferrer"`
- **Files:** page.tsx:513-571

**No Major Concerns:**
- No user input handling (except roster search which is client-side only)
- No API routes with secrets
- No cookies or local storage
- innerHTML usage is with static content only

## Accessibility Analysis

**Rating:** B-

**Strengths:**
- Semantic HTML (`<nav>`, `<header>`, `<section>`, `<footer>`)
- `aria-label` on mobile menu button
- `aria-expanded` on FAQ buttons and mobile menu
- `lang="en"` on html element

**Issues:**

**[P3] Missing ARIA on Social Icons**
- SVG icons in social links need `aria-hidden="true"`
- Social link `<a>` tags have `aria-label` but inconsistently

**[P3] No Skip Navigation Link**
- No "skip to content" link for keyboard users

**[P4] Color Contrast**
- `--text-dark: #71717A` on `--bg-main: #09090B` — may not meet WCAG AA (4.5:1 ratio)
- Needs verification with contrast checker
